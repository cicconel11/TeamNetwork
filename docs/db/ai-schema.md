# AI Subsystem Schema

**Scope:** The database surface backing the TeamNetwork AI assistant — conversation, observability/governance, the RAG / embedding pipeline, the semantic cache, and the confirmation-gated mutation layer.

**Source of truth:** `apps/web/src/types/database.ts` (generated Supabase types). For runtime/architecture detail see the OKF bundle in [`docs/agent/`](../agent/index.md) (start at `docs/agent/index.md`); for per-table column dumps see `docs/db/okf/ai_*.md`. This doc is the hand-maintained narrative that ties the tables together and is summarized (not duplicated) in [`schema-audit.md`](./schema-audit.md).

> **Freshness note.** As of the last refresh, `database.ts` (and therefore the generated `docs/db/okf/` bundle) had **not** been regenerated after `20261224000000_knowledge_documents.sql`, so `knowledge_documents` is documented here from its migration but is missing from the generated types and OKF index. Run `gen:types` then `bun run gen:db-okf` to close that gap.

## Table inventory

Eleven `ai_*` tables exist in `database.ts`, plus `knowledge_documents` (a RAG source created by a later migration). Note: `ai_chunks` is **not** a table — the chunk table is `ai_document_chunks`; `replace_ai_chunks` is an RPC.

| Group                      | Tables                                                                                      |
| -------------------------- | ------------------------------------------------------------------------------------------- |
| Conversation core          | `ai_threads`, `ai_messages`, `ai_feedback`                                                  |
| Observability / governance | `ai_audit_log`, `ai_spend_ledger`                                                           |
| Semantic cache             | `ai_semantic_cache`                                                                         |
| RAG / embedding pipeline   | `ai_embedding_queue`, `ai_document_chunks`, `knowledge_documents`, `ai_indexing_exclusions` |
| Mutation / action layer    | `ai_pending_actions`, `ai_draft_sessions`                                                   |

## Table-by-table reference

### Conversation core

| Table         | Purpose                                                                                                                             | Key columns                                                                                                                                                                                 | FKs / relations                                                                                                                                                                 |
| ------------- | ----------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ai_threads`  | Conversation containers, scoped to user + org + `surface` (`general`/`members`/`analytics`/`events`). Soft-delete via `deleted_at`. | `id`, `user_id`, `org_id`, `surface`, `title`, `metadata`, `deleted_at`                                                                                                                     | → `organizations`. Cascade-deletes on `auth.users` deletion.                                                                                                                    |
| `ai_messages` | Individual turns within a thread. Denormalized `user_id`/`org_id`.                                                                  | `id`, `thread_id`, `org_id`, `user_id`, `role`, `content`, `status` (`pending`/`streaming`/`complete`/`error`), `intent`, `intent_type`, `context_surface`, `tool_calls`, `idempotency_key` | **Composite FK** `(thread_id, user_id, org_id)` → `ai_threads(id, user_id, org_id)` — enforces ownership at the DB level. Table-wide unique partial index on `idempotency_key`. |
| `ai_feedback` | Per-message thumbs/ratings on assistant responses.                                                                                  | `id`, `message_id`, `user_id`, `rating`, `comment`                                                                                                                                          | → `ai_messages`. RLS: users read/write own; admins read within org. Feeds `npm run evals:ai:feedback`.                                                                          |

### Observability / governance

| Table             | Purpose                                                                                                              | Key columns                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    | FKs / relations                                                                                                                                                                                                                                                                                                                   |
| ----------------- | -------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ai_audit_log`    | One row per AI request. Service-role only (RLS enabled, **zero policies**). 33 columns; 90-day TTL via `expires_at`. | tokens (`input_tokens`/`output_tokens`), `latency_ms`, `model`, `intent`/`intent_type`, cache (`cache_status`, `cache_entry_id`, `cache_bypass_reason`), context (`context_surface`, `context_token_estimate`), RAG (`rag_chunk_count`, `rag_grounded`, `rag_grounding_mode`, `rag_top_similarity`, `rag_grounding_failures`, `rag_grounding_latency_ms`, `rag_error`), safety (`safety_verdict`, `safety_categories`, `safety_latency_ms`), `stage_timings` (JSONB), `write_action_id`, `write_action_status` | **No FK** on `user_id`/`org_id` — intentional, so the audit row survives user/org deletion (`user_id` set NULL on account deletion). | Table               | Purpose                                                        | Key columns                                                                                                                                                                                                   | FKs / relations                                                                                                                                                                                                                                                                                                                                                     |
| ------------------- | -------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ai_semantic_cache` | Exact-hash (SHA-256) prompt→response cache. Service-role only. | `org_id`, `surface`, `permission_scope_key`, `cache_version`, `prompt_hash`, `prompt_normalized`, `response_content` (≤16k chars), `expires_at`, `invalidated_at`, `invalidation_reason`, `source_message_id` | → `organizations` (CASCADE); `source_message_id` → `ai_messages` (SET NULL). ### Mutation / action layer

The assistant never mutates data directly — every write goes through a server-owned, confirmation-gated action.

| Table                | Purpose                                                                                                          | Key columns                                                                                                                                                                                     | FKs / relations                                                                                                                                                                                        |
| -------------------- | ---------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `ai_pending_actions` | Server-owned confirmation-gated write actions.                                                                   | `org_id`, `user_id`, `thread_id`, `action_type`, `payload`, `previous_payload`, `status`, `revise_count`, `result_entity_type`/`result_entity_id`, `executed_at`, `error_message`, `expires_at` | → `ai_threads`, `organizations`. RLS: user+org scoped; admins access only their own.                                                                                                                   |
| `ai_draft_sessions`  | Active per-thread multi-turn draft continuation state (collects missing fields across turns). Service-role only. | `organization_id`, `user_id`, `thread_id`, `draft_type`, `draft_payload`, `missing_fields[]`, `status`, `pending_action_id`, `expires_at`                                                       | → `ai_pending_actions`, `ai_threads`, `organizations`. `draft_type` CHECK covers create/update/delete for announcement, job_posting, event + send chat/group message + create discussion reply/thread. |

## Runtime data flow

**Chat turn — `threads → messages → audit → spend`.**
`POST /api/ai/[orgId]/chat` → auth (`getAiOrgContext`, admin-only) → message-safety assessment → `init_ai_chat()` RPC creates/reuses an `ai_threads` row and inserts the user `ai_messages` row (skipped via `p_skip_user_message` when the turn is a terminal refusal — see migration `20261211000000`). A `TurnExecutionPolicy` decides cache/RAG/tool eligibility. The assistant message streams over SSE, is finalized in `ai_messages`, and every turn writes one `ai_audit_log` row (tokens, latency, cache status, RAG grounding, safety, `stage_timings`). Provider cost is charged to `ai_spend_ledger` via `charge_ai_spend()`; a pre-call gate (`get_ai_spend_for_period()`) enforces the monthly cap. Users can rate any assistant message → `ai_feedback`.

**RAG / embedding — `queue → chunks / knowledge_documents → search`.**
Triggers (`enqueue_ai_embedding()`) on the 8 source tables push rows into `ai_embedding_queue` on insert / meaningful update / soft-delete. A worker embeds content into `ai_document_chunks` (768-dim vectors), respecting `ai_indexing_exclusions`. `knowledge_documents` is the admin-curated 8th source flowing through the same pipeline. Retrieval uses the `search_ai_documents()` RPC with an optional `p_audience_filter` (added `20261220000000`) so audience-restricted chunks aren't grounded for the wrong role.

**Semantic cache.** Sits between message persistence and the LLM call. Only standalone first-turn `general` prompts are looked up by exact SHA-256 hash in `ai_semantic_cache`; misses use `shared_static` context and skip RAG; writes happen post-stream. Hits/misses are recorded in `ai_audit_log.cache_status`; an hourly cron purges via `purge_expired_ai_semantic_cache()`.

**Mutation / action layer.** A `prepare_*` tool validates a draft → `ai_pending_actions` row + `pending_action` SSE event. Multi-turn field collection persists in `ai_draft_sessions` (linked via `pending_action_id`). A separate confirm/cancel route executes the action server-side. `ai_audit_log.write_action_id`/`write_action_status` link audit rows to the action.

## Intentional divergences

- **`ai_audit_log.user_id` / `org_id`** — `uuid NOT NULL` with no FK. Audit must survive user/org deletion; CASCADE would destroy the audit trail and SET NULL conflicts with NOT NULL. Service-role-only (RLS enabled, zero policies).
- **`ai_messages` composite FK** — `(thread_id, user_id, org_id)` → `ai_threads(id, user_id, org_id)` instead of a simple `thread_id` FK, so RLS doesn't need subquery joins for ownership and `user_id`/`org_id` can't drift from the parent thread.
- **`ai_messages.idempotency_key` uniqueness** is table-wide, not scoped to `(org_id, user_id)` — client-generated v4 UUIDs must be globally unique for exactly-once delivery across the cluster.

## Recent AI migrations

| Migration                                              | Change                                                                                                                                                                                         |
| ------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `20261205000000_ai_draft_sessions_add_mutation_drafts` | Adds update/delete `draft_type` variants (announcement/job_posting/event) so update/delete continuations persist across turns.                                                                 |
| `20261211000000_ai_init_skip_user_message`             | Adds `p_skip_user_message` to `init_ai_chat()`; refused turns no longer persist the user message (avoids storing refused PII/jailbreak content) but still create the thread.                   |
| `20261220000000_search_ai_documents_audience`          | Adds `p_audience_filter text[]` to `search_ai_documents()`; closes a gap where RAG retrieval ignored audience restrictions. Drops the old 5-arg overload.                                      |
| `20261223000000_ai_chunks_source_table_check_sync`     | Data-integrity fix: rebuilds `ai_document_chunks_source_table_check` to the sources the chunker actually inserts (adds `mentor_profiles`, `form_submissions`).                                 |
| `20261224000000_knowledge_documents`                   | Creates `knowledge_documents` (admin RLS, org-scoped index, `updated_at` trigger); extends the source_table CHECK to 8 sources, `enqueue_ai_embedding()`, and `backfill_ai_embedding_queue()`. |
