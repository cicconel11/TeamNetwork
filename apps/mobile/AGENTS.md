# Mobile App Guidelines

These notes apply only to `apps/mobile/`.

## Project Structure

- `app/` contains Expo Router routes and layouts.
- `app/(auth)/` contains unauthenticated flows: welcome, login, signup, callback, forgot/reset password.
- `app/(app)/` mounts the authenticated app shell with `OrgProvider` and the app drawer.
- `app/(app)/(drawer)/` contains org list, profile, terms, delete-account, and the org-scoped route tree.
- `app/(app)/(drawer)/[orgSlug]/(tabs)/` contains the primary tab loop: home, calendar, announcements, members, menu. `alumni` exists but is hidden from the tab bar.
- `app/(app)/(drawer)/[orgSlug]/` contains feature stacks like chat, events, workouts, schedules, forms, jobs, donations, parents, notifications, settings, and search.
- `src/components/` contains reusable UI and feature components.
- `src/contexts/` contains auth, org, network, and color-scheme providers.
- `src/hooks/` contains data hooks and app behavior hooks.
- `src/lib/` contains Supabase, analytics, notifications, uploads, tokens, and shared helpers.
- `src/navigation/` contains `AppDrawer.tsx` and `DrawerContent.tsx`.
- `assets/` contains icons, splash, and brand artwork.
- `plugins/` contains Expo config plugins. `app.json` is the source of truth for native config.
- `android/` and `ios/` are generated by `expo prebuild` and are gitignored. Do not treat them as hand-maintained source files.

## Commands

Run commands from `apps/mobile/`.

- `bun run start`: Expo dev server.
- `bun run start:dev-client`: Expo dev server for a development build.
- `bun run ios`: Launch iOS dev client.
- `bun run android`: Launch Android dev client with local SDK env setup.
- `bun run web`: Launch Expo web.
- `bun run prebuild`: Generate native projects.
- `bun run prebuild:clean`: Regenerate native projects from scratch.
- `bun run run:ios`: Build and run locally on iOS.
- `bun run run:android`: Build and run locally on Android.
- `bun run config`: Print resolved Expo config.
- `bun run config:introspect`: Print introspected Expo config.
- `bun run android:doctor`: Verify Android SDK/Java/adb setup.
- `bun run typecheck`: TypeScript check.
- `bun run test`: Jest tests.
- `bun run test:watch`: Jest watch mode.
- `bun run test:coverage`: Coverage run.
- `bun run test:ci`: CI-style Jest run.

## Architecture

- Root provider order is `ColorSchemeProvider -> AuthProvider -> NetworkProvider -> GestureHandlerRootView -> ToastProvider -> Stack`.
- Auth and deep-link handling live in `app/_layout.tsx`.
- `app/(app)/_layout.tsx` wraps the authenticated shell in `OrgProvider` and mounts `AppDrawer`.
- The drawer uses `drawerType: "front"` and a custom `DrawerContent`.
- Org routes are guarded in `app/(app)/(drawer)/[orgSlug]/_layout.tsx`; invalid or unloaded org state redirects back to `/(app)`.
- Styling is based on `useThemedStyles`, `src/lib/design-tokens.ts`, `src/lib/typography.ts`, and `src/lib/chrome.ts`.
- Auth uses Supabase with AsyncStorage-backed persistence and PKCE.
- Analytics live under `src/lib/analytics/`.
- Push notifications live in `src/lib/notifications.ts` and `src/hooks/usePushNotifications.ts`.
- Donations and some admin flows bridge to the web app via `src/lib/web-api.ts`.

## Coding Conventions

- TypeScript strict mode, 2-space indentation, semicolons, double quotes.
- Prefer Expo Router patterns over ad hoc navigation wiring.
- Prefer `Pressable`, `expo-image`, `SafeAreaView`, and `LinearGradient` to match existing code.
- Prefer `useThemedStyles` and design tokens over inline colors or legacy theme helpers for new work.
- Keep Supabase nullability explicit in UI code.
- Use `undefined`, not `null`, for optional RPC parameters.
- If you edit route files with parentheses or brackets in the path, quote the shell path.

## Testing

- Tests live in `__tests__/`.
- Jest is configured in `jest.config.js` with a Node environment and Expo Babel transform.
- The app currently focuses on pure-function and lib-level tests; many screen and native interaction flows are better validated with dev builds and manual/device testing.
- If Jest fails in a sandbox because of Watchman, note that explicitly rather than papering over it.

## Config Notes

- `.env.local` is gitignored. Use `.env.example` as the template.
- `EXPO_PUBLIC_SUPABASE_URL`, `EXPO_PUBLIC_SUPABASE_ANON_KEY`, and `EXPO_PUBLIC_WEB_URL` are the core runtime vars.
- Optional env vars currently cover PostHog, Sentry, hCaptcha, and Google OAuth-related config.
- Native permissions and plugins are configured in `app.json`.
- If a native change is required, prefer updating `app.json` or `plugins/` and regenerating with `expo prebuild` rather than hand-editing generated native folders.

## Agent Notes

- Use `rg` for search.
- Use `bun` commands here, not `npm`.
- When documenting navigation, refer to actual Expo Router paths under `app/`, not web `src/app/` paths.
